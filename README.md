# Intelligent Content API

A RESTful API service that allows users to upload text content, which is then processed by an AI service to generate summaries and sentiment analysis.

## Features

- **User Authentication**: JWT-based signup and login.
- **Content Management**: Upload, list, retrieve, and delete text content.
- **AI Integration**: Automatic summarization and sentiment analysis (Mocked for demo, extensible).
- **Async Processing**: Non-blocking AI processing.
- **Dockerized**: Easy to run with Docker Compose.

## Setup Instructions

### Prerequisites

- Docker and Docker Compose
- Python 3.9+ (for local development)

### Running with Docker

1.  Build and run the containers:
    ```bash
    docker-compose up --build
    ```
2.  Access the API documentation at `http://localhost:8000/docs`.

### Running Locally

1.  Create a virtual environment:
    ```bash
    python -m venv venv
    source venv/bin/activate  # On Windows: venv\Scripts\activate
    ```
2.  Install dependencies:
    ```bash
    pip install -r requirements.txt
    ```
3.  Run the application:
    ```bash
    uvicorn app.main:app --reload
    ```

## API Documentation

The API documentation is automatically generated by FastAPI and can be accessed at `/docs` (Swagger UI) or `/redoc`.

### Endpoints

-   `POST /signup`: Register a new user.
-   `POST /login`: Login and get an access token.
-   `POST /contents`: Upload text content.
-   `GET /contents`: List user's content.
-   `GET /contents/{id}`: Get specific content details.
-   `DELETE /contents/{id}`: Delete content.

## Design Decisions

-   **FastAPI**: Chosen for its speed, automatic validation, and async support.
-   **PostgreSQL**: Robust relational database for structured data.
-   **SQLAlchemy**: ORM for database interactions.
-   **Modular Architecture**: Separation of concerns (API, Core, Models, Schemas, Services).

## GCP Architecture (Theoretical)

To deploy this on Google Cloud Platform:

1.  **Container Registry**: Push the Docker image to Google Container Registry (GCR) or Artifact Registry.
2.  **Cloud Run**: Deploy the container to Cloud Run for a serverless, scalable API.
3.  **Cloud SQL**: Use a managed PostgreSQL instance for the database.
4.  **Secret Manager**: Store sensitive configuration (DB credentials, API keys) in Secret Manager.
5.  **CI/CD**: Use Cloud Build or GitHub Actions to build and deploy on push to main.
